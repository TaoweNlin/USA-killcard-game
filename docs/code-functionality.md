# 功能明细

本文按当前源码记录已经实现的能力。

## 数据

- `src/data/cards.ts`：定义标准 + EX 108 张牌，包含基础牌、锦囊/延时锦囊和装备牌。
- `src/data/characters.ts`：定义 22 名中文角色、阵营、体力、性别、技能和角色图。
- `src/data/roles.ts`：定义总统、幕僚、反对、资本，以及 5 人身份配置。
- `src/data/assets.ts`：生成运行图片路径并保留 `src`、`thumb`、`alt`、`focalPoint`、`credit`、`license` 字段。
- `src/data/types.ts`：共享类型，覆盖牌、角色、玩家状态、响应窗口、选择窗口、动作和视觉事件。

## 规则引擎

`AmericaKillGame` 负责完整牌局状态：

- 开局：玩家可指定本人角色/身份，AI 自动补齐角色和身份。
- 阶段：准备、判定、摸牌、出牌、弃牌、结束、游戏结束。
- 牌堆：摸牌、弃牌、洗回弃牌堆和平局兜底。
- 牌效果：基础牌、核心锦囊、延时锦囊和标准装备的主要结算。
- 响应：支持【洗】、【票】、【事实核查】和部分视为打出。
- 选择：支持目标区域牌、亮出牌、弃置代价、武器触发和临场拼稿。
- 死亡与胜负：濒死救援、身份奖惩、总统/反对/资本/幕僚胜利条件。
- AI：能自动推进回合，并在需要人类响应或选择时暂停。

## UI

`src/main.ts` 使用稳定 DOM 区域渲染：

- 开局角色/身份选择界面。
- 五人围桌角色牌、身份圆章、血点、装备条、判定区和技能入口。
- 底部手牌区、操作按钮条、目标选择、响应窗口和通用选择弹层。
- 右上轮次/牌堆信息、菜单、日志弹窗、卡牌详情弹窗和胜负弹窗。
- `#visual-layer` 承载出牌、响应、连线、判定、伤害、回血和死亡表现。

## 样式

`src/styles.css` 实现：

- 深蓝美国政治竞技场背景。
- 角色卡、手牌、卡牌详情、装备文字条和判定小卡。
- 可用牌、选中牌、目标、当前行动、濒死和死亡状态。
- 视觉事件动画和弹字。
- 桌面优先布局，并保留基础响应式规则。

## 测试

- `src/tests/data-validation.ts`：校验 108 张牌、4 张 EX、分类数量、22 名角色、身份池、技能覆盖和资源路径契约。
- `src/tests/rules-smoke.ts`：覆盖开局、响应、事实核查、弃牌、目标选择、判定、濒死救援、公开募资、借号开火、临场拼稿、装备触发和视觉事件等关键路径。

## 调试接口

浏览器环境暴露：

- `window.americaKillGame`
- `window.render_game_to_text()`
- `window.advanceTime(ms)`

这些接口服务于开发和 QA，不承诺作为稳定外部 SDK。

## 发布边界

- `public/assets` 是运行时资源，会随 GitHub 仓库发布。
- `output/`、`dist/`、`node_modules/`、`progress.md` 和 `design-qa.md` 属于本地开发/构建/证据文件，不进入仓库。
